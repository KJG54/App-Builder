# Documentation Standards

Status: Draft

## General

- Every project must have a README
- Every project must have a CLAUDE.md
- Architecture must be versioned and current

## Code Comments

- Default: no comments
- Only add a comment when the WHY is non-obvious
- Never describe what the code does — name the function well instead

## ADRs

- Required for every significant technology or architecture decision
- Use categorized naming: ADR-ARCH, ADR-SEC, ADR-DATA, ADR-INFRA, ADR-API, ADR-INT
